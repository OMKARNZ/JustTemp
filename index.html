FreakBeast Fitness Tracker - Project Abstraction

1. Project Overview

FreakBeast Fitness Tracker is a modern, cross-platform Android application developed using a MERN-adjacent stack. It is designed to assist users in managing their workout routines, tracking real-time session performance, establishing fitness goals, and analyzing comprehensive progress reports.

2. Core Functional Modules (Features)

User Authentication & Profile: Secure sign-up/login using JWT, along with profile management (e.g., BMI calculation, user stats).
Workout Planner: Interface to create, modify, and view flexible daily and weekly workout routines with granular exercise details (sets, reps, weight, rest times).
Session Tracking: Real-time workout session logging, allowing users to actively record set metrics as they train.
Goal Management: Tools to set up and track specific, measurable fitness targets (e.g., weight loss, strength gains) with visual progress indicators.
Progress Reporting: Historical summaries of workouts and the ability to generate and download comprehensive PDF progress reports.
Customization/Configuration: Support for Light/Dark mode switching and notification controls.

3. Technology Stack

Frontend / Client UI: React, Vite Language : TypeScript (end-to-end type safety)
Styling: Tailwind CSS & shadcn-ui (utility-first and component libraries)
Backend / Logic : Node.js via Supabase Edge Functions (for complex rules, PDF generation) Database & Auth : Supabase / PostgreSQL (handling data persistence, real-time subscriptions, secure auth, and Row Level Security)

4. Architectural Design

The platform architecture utilizes a clear separation of concerns, divided into three main layers:
1. Client Layer: Manages views, state management, and user interaction within the React environment. Employs client-side caching to reduce unnecessary latency and redundant API calls. 2. Logic Layer: Contains business rules executed serverlessly via Supabase Edge Functions. It runs complex calculations securely.
3. Data Persistence Layer: Ensures structured, relational, and real-time data integrity via PostgreSQL, governed by strict authorization boundaries using Row Level Security (RLS). Communications are strictly authenticated through Supabase APIs.

5. Development Principles

Modular Components: Emphasis on reusable functional components to reduce code duplication. Strict Typing : TypeScript enforces data contracts across all layers, minimizing runtime errors.
Optimized Data Fetching : Responsive UI achieved by managing data flow efficiently and caching localized state.
